MathWriting: A Dataset For Handwritten Mathematical Expression Recognition
MathWriting：用于手写数学表达式识别的数据集

Philippe Gervais pgervais@acm.org InceptiveSwitzerland ,  Anastasiia Fadeeva fadeich@google.com Google DeepMindZurichSwitzerland  and  Andrii Maksai amaksai@google.com Google DeepMindZurichSwitzerland
(23 February 2025)
Abstract.  摘要。

Recognition of handwritten mathematical expressions allows to transfer scientific notes into their digital form. It facilitates the sharing, searching, and preservation of scientific information. We introduce MathWriting, the largest online handwritten mathematical expression dataset to date. It consists of 230k human-written samples and an additional 400k synthetic ones. This dataset can also be used in its rendered form for offline HME recognition. One MathWriting sample consists of a formula written on a touch screen and a corresponding LaTeX expression. We also provide a normalized version of LaTeX expression to simplify the recognition task and enhance the result quality. We provide baseline performance of standard models like OCR and CTC Transformer as well as Vision-Language Models like PaLI on the dataset. The dataset together with an example colab is accessible on Github.
手写数学表达式识别能够将科学笔记转化为数字形式。它促进了科学信息的共享、搜索和保存。我们介绍了 MathWriting，这是迄今为止最大的在线手写数学表达式数据集。它包含 230k 个人书写的样本和额外的 400k 个合成样本。该数据集也可以以渲染形式用于离线手写数学表达式识别。一个 MathWriting 样本包括在触摸屏上书写的公式和一个相应的 LTeX 表达式。我们还提供了 LTeX 表达式的标准化版本，以简化识别任务并提高结果质量。我们还提供了标准模型（如 OCR 和 CTC Transformer）以及视觉语言模型（如 PaLI）在数据集上的基线性能。该数据集和示例 colab 可以在 Github 上获取。

Online text recognition interprets handwritten input on devices with touchscreens like smartphones and tablets. This process analyzes the handwriting as a sequence of strokes, rather than a static image (Graves2009ANC). The recognition of mathematical expressions (MEs) presents a significant challenge, and has historically received less research attention compared to character and word recognition (ALTaee2020HandwrittenRA). The fundamental differences between mathematical expression (ME) recognition and text recognition create obstacles to the direct application of improvements developed in one domain to the other. While mathematical expressions (MEs) utilize many of the same symbols as standard text, they are distinguished by a more rigid and two-dimensional structural organization, see Figure 1. Where text can be treated to some extent as a one-dimensional problem amenable to sequence modeling, MEs require two-dimensional analysis due to the importance of symbol spatial positioning. It is also different from symbol segmentation or object detection because the output of a recognizer has to contain the relationship between symbols, serialized in some form (LaTeX, a graph, InkML, etc.). Handwritten mathematical expressions (HMEs), like handwritten text, are more challenging to recognize than their printed counterparts due to increased variability in writing and data scarcity.
在线文本识别解释在带触摸屏的设备（如智能手机和平板电脑）上的手写输入。该过程将手写内容分析为一系列笔划，而不是静态图像（Graves2009ANC）。数学表达式（ME）的识别是一个重大挑战，与字符和单词识别相比，历史上获得的研究关注较少（ALTaee2020HandwrittenRA）。数学表达式（ME）识别与文本识别之间的基本差异，阻碍了在一个领域开发的改进直接应用于另一个领域。虽然数学表达式（ME）使用了许多与标准文本相同的符号，但它们的特点是更严格和二维的结构组织，见图 1。文本在一定程度上可以被视为一维问题，适合序列建模，而数学表达式（ME）由于符号空间定位的重要性，需要二维分析。 它也与符号分割或目标检测不同，因为识别器的输出必须包含符号之间的关系，以某种形式序列化（LTeX、图、InkML 等）。手写数学表达式（HMEs），与手写文本一样，由于其书写变化增加和数据稀缺，识别难度比印刷版本更大。

Handwritten data acquisition is expensive due to the need for manual input with specialized hardware like touchscreens and digital pens. The publication of the MathWriting dataset aims to address the data requirements for mathematical expression recognition research. Samples include a large number of human-written inks, as well as synthetic ones. MathWriting can readily be used with other online datasets like CROHME (crohme) or Detexify (detexifydataset) - we publish the data in InkML format to facilitate this. It can also be used for offline ME recognition by rasterizing the inks, using code provided on the Github page111https://github.com/google-research/google-research/tree/master/mathwriting
手写数据采集成本高昂，因为需要使用触摸屏和数字笔等专用硬件进行手动输入。发布 MathWriting 数据集旨在解决数学表达式识别研究的数据需求。样本包括大量人工书写的墨迹，以及合成的墨迹。MathWriting 可以方便地与其他在线数据集（如 CROHME（crohme）或 Detexify（detexifydataset））一起使用——我们将数据以 InkML 格式发布以促进这一过程。它也可以通过在 Github 页面上提供的代码将墨迹栅格化，用于离线数学表达式识别。.

MathWriting is the largest set of online HME published so far - both human-written and synthetic. It significantly expands the set of symbols covered by CROHME (crohme), enabling more sophisticated recognition capabilities. Since inks can be rasterized, MathWriting can also been seen as larger than existing offline HME datasets (deng2017imagetomarkup; yuan2022syntax; aidadataset). For these reasons we introduce a new benchmark, applicable to both online and offline ME recognition.

2. Related work  2. 相关工作

Mathematical expression recognition offers valuable tools for educational software, including features like Math Notes on iPhone (MathNotes). As tablets such as the Google Pixel, iPad, and Remarkable gain widespread adoption, the demand for effective handwriting tools is growing.
数学表达式识别为教育软件提供了有价值的工具，包括 iPhone 上的数学笔记（MathNotes）等功能。随着 Google Pixel、iPad 和 Remarkable 等平板电脑的广泛采用，对有效手写工具的需求正在增长。

There are multiple datasets, such as Aida Calculus Math Handwriting Recognition Dataset (aidadataset) and IM2LATEX-100K (IM2LATEX), that focus on image-based (offline) math recognition. The Aida dataset consists solely of synthetically generated handwritten math expressions. IM2LATEX-100K has become a key benchmark for assessing performance in typeset math formula recognition. There is also a dataset for typeset math formula detection – TFD-ICDAR 2019 (Mahdavi2019ICDAR2C) that was part of Competition on Recognition of Handwritten Mathematical Expressions (CROHME) 2019. The most widely used dataset for evaluating online math recognition systems originates from the CROHME competition (crohme). We establish in Section 4.4 that the MathWriting dataset surpasses CROHME with a richer vocabulary and more unique formulas.
存在多个专注于基于图像（离线）的数学识别的数据集，例如 Aida Calculus Math Handwriting Recognition Dataset（aidadataset）和 IM2LATEX-100K（IM2LATEX）。Aida 数据集完全由合成生成的手写数学表达式组成。IM2LATEX-100K 已成为评估排版数学公式识别性能的关键基准。还有一个用于排版数学公式检测的数据集——TFD-ICDAR 2019（Mahdavi2019ICDAR2C），它是 2019 年手写数学表达式识别竞赛（CROHME）的一部分。用于评估在线数学识别系统最广泛使用的数据集源自 CROHME 竞赛（crohme）。我们在第 4.4 节中建立，MathWriting 数据集在词汇更丰富、独特公式更多方面超越了 CROHME。

The highest-performing model in the CROHME 2019 competition (Mahdavi2019ICDAR2C) was an LSTM-based encoder-decoder, achieving its result through training on an augmented dataset. This highlights the critical role of training data volume and methods for synthetic data generation. The MyScript model (myscript), a CTC BLSTM architecture, achieved third place in the competition. Similarly, Section 5.2 reveals that a CTC Transformer yields strong results on the MathWriting dataset. We also present results obtained with the encoder-decoder model PaLI (chen2023pali).
CROHME 2019 竞赛（Mahdavi2019ICDAR2C）中表现最佳的模型是基于 LSTM 的编码器-解码器，其通过在增强数据集上训练取得该结果。这突显了训练数据规模和合成数据生成方法的关键作用。MyScript 模型（myscript），一种 CTC BLSTM 架构，在竞赛中获得了第三名。类似地，第 5.2 节揭示 CTC Transformer 在 MathWriting 数据集上也能取得优异结果。我们还展示了基于编码器-解码器模型 PaLI（chen2023pali）获得的结果。

3. Dataset Creation  3.数据集创建

MathWriting dataset primarily consists of LaTeX expressions from Wikipedia, more details about the acquisition of expressions are provided in Appendix B. These expressions were used for both ink collection from human contributors Section 3.1 as well as synthetic data generation Section 3.2. We did a very limited filtering of very noisy human-written examples (described in Appendix C).
MathWriting 数据集主要由维基百科的 LTeX 表达式组成，关于表达式获取的更多细节在附录 B 中提供。这些表达式既用于人类贡献者的墨水收集（第 3.1 节），也用于合成数据生成（第 3.2 节）。我们对非常嘈杂的人类书写示例进行了非常有限的过滤（描述在附录 C 中）。

3.1. Ink Collection  3.1.墨水收集

Inks were obtained from human contributors through an in-house Android app. Participants agreed to the standard Google terms of use and privacy policy. The task consisted in copying a rendered mathematical expression (prompt) shown on the device’s screen using either a digital pen or a finger on a touch screen. Mathematical expressions used as prompt were first obtained in LaTeX format, then rendered into a bitmap through the LaTeX compiler (see Appendix A for the template used). 95% of MathWriting expressions were obtained from Wikipedia. The remaining ones were generated to cover underrepresented cases in Wikipedia, like isolated letters with nested sub/superscripts or complicated fractions (see Section B). Contributors were hired internally at Google. 6 collection campaigns were run between 2016 and 2019, each lasting between 2 to 3 weeks. Collected data contains only inks and labels, so no personally identifiable information is present in the dataset. Offensive content is highly unlikely because LaTeX expressions were taken from Wikipedia and we conducted a filtering of noisy data (described in Appendix C).
墨迹数据通过内部安卓应用程序从人类贡献者那里获取。参与者同意了标准的谷歌使用条款和隐私政策。任务包括使用数字笔或触摸屏上的手指，复制显示在设备屏幕上的渲染后的数学表达式（提示）。用作提示的数学表达式首先以 LTeX 格式获取，然后通过 LTeX 编译器渲染为位图（有关使用的模板，请参见附录 A）。MathWriting 表达式的 95%来自维基百科。其余的表达式是为了涵盖维基百科中未充分代表的情况，例如带有嵌套下标/上标的孤立字母或复杂的分数（参见第 B 节）。贡献者是在谷歌内部招聘的。在 2016 年至 2019 年间进行了 6 次收集活动，每次持续 2 到 3 周。收集的数据仅包含墨迹和标签，因此数据集中不存在可识别个人身份的信息。由于 LTeX 表达式来自维基百科，并且我们进行了噪声数据的过滤（描述在附录 C 中），因此攻击性内容极不可能存在。

3.2. Synthetic Samples and Isolated Symbols
3.2.合成样本和孤立符号

We created synthetic samples in order to further increase the label diversity for training. This also enabled compensating for limitations of the human collection like the maximum length of the expressions, which were limited by the size of the screen they were written on. We used LaTeX expressions from Wikipedia that were not used in the data collection. The resulting synthetic data has a 90th percentile of expression length of 68 characters, compared to 51 in train. This is especially important as deep neural nets often fail to generalize to inputs longer than their training data (anil2022exploring; Varis_2021). Using synthetic long inks together with the original human-written inks can help to eliminate that problem as shown in (Timofeev_2023; narayanan2019recognizing). The synthesis technique is as follows: starting from a raw LaTeX mathematical expression, we computed a DVI file using the LaTeX compiler, from which we extracted bounding boxes. We then used those bounding boxes to place handwritten individual symbols, resulting in a complete expression. See Figure 2 for an example of extracted bounding boxes and the resulting synthetic example. We make the bounding box data public to facilitate custom synthetic dataset creation.
我们创建了合成样本，以进一步增加训练的标签多样性。这也使我们能够弥补人工收集的局限性，例如表达式最大长度受限于书写屏幕的大小。我们使用了维基百科上未被数据收集使用的 LTeX 表达式。生成的合成数据中，表达式的 90 分位数长度为 68 个字符，而训练集中的为 51 个。这尤其重要，因为深度神经网络往往无法泛化到比其训练数据更长的输入（anil2022exploring; Varis_2021）。使用合成长墨迹与原始人工书写的墨迹相结合，可以帮助消除该问题，如(Timofeev_2023; narayanan2019recognizing)所示。合成技术如下：从原始 LTeX 数学表达式开始，我们使用 LTeX 编译器计算生成 DVI 文件，从中提取边界框。然后我们使用这些边界框放置手写单个符号，从而生成完整表达式。参见图 2，以展示提取的边界框和生成的合成示例。我们将边界框数据公开，以促进自定义合成数据集的创建。

Inks for individual symbols are all from the symbols split. They have been manually extracted from inks in train. For each symbol, we manually selected strokes corresponding to it for 20-30 distinct occurrences in train, and used that information to generate a set of individual inks. Similar synthesis techniques have been used by (crohme) with inks, (deng2017imagetomarkup) and (aidadataset) with raster images.
单个符号的墨水均来自符号分割。它们被手动从训练集中的墨水中提取。对于每个符号，我们在训练集中手动选择了对应它的笔画，共 20-30 个不同出现，并使用这些信息生成一组单个墨水。类似合成技术已被(crohme)用于墨水，(deng2017imagetomarkup)和(aidadataset)用于光栅图像。

The synthetic inks can incorporate symbols from multiple participants which can result in statistical inconsistency. However, even human handwriting displays occasional stylistic variations within one sample (see Appendix J.1 ink 02229⁢a⁢0⁢c⁢174⁢d⁢8⁢d⁢b⁢e02229𝑎0𝑐174𝑑8𝑑𝑏𝑒02229a0c174d8dbe02229 italic_a 0 italic_c 174 italic_d 8 italic_d italic_b italic_e). We argue that having more diversity in synthetic dataset is valuable for training robust recognition models. We also show in ablation experiments (Section 5.3) that synthetic data improves the quality of recognition.
合成墨水可以包含来自多个参与者的符号，这可能导致统计不一致。然而，即使是人类手写在一份数据中也偶尔显示风格变化（参见附录 J.1 墨水 02229⁢a⁢0⁢c⁢174⁢d⁢8⁢d⁢b⁢e022290174802229a0c174d8dbe02229 italic_a 0 italic_c 174 italic_d 8 italic_d italic_b italic_e ）。我们认为在合成数据集中拥有更多多样性对训练鲁棒的识别模型是有价值的。我们还通过消融实验（第 5.3 节）表明合成数据提高了识别质量。

Another important distinction between the synthetic and human-written inks is the stroke order. For synthetic inks, stroke order follows the order of the bounding boxes in the DVI file, which can be different from the usual order of writing for mathematical expressions. However, the writing order within a given symbol is consistent with human writing.
合成墨迹和人类书写墨迹之间的另一个重要区别是笔顺。对于合成墨迹，笔顺遵循 DVI 文件中边界框的顺序，这可能与数学表达式通常的书写顺序不同。然而，在给定符号内部的书写顺序与人类书写一致。

3.3. Dataset Split  3.3.数据集划分

MathWriting is composed of five different sets of samples, which we call ’splits’: train, valid, test, symbols, and synthetic. The splits train, valid and test consist only of human-written examples. The split symbols is provided for synthetic data generation and is not used in training. The split of human-written samples between train, valid and test was partially done based on writers, partially based on labels. More details are provided in Appendix K. Experiments have shown that a more important factor than the handwriting style was whether the label had already been seen during training. This fact is also supported by research in the area of compositional generalization (keysers2020measuring). In the published version, valid has a 55% (8.5k samples) intersection with train based on unique normalized labels, and test has an 8% intersection (647 samples). We chose to have a low intersection between train and test in order to correctly measure generalization of trained models to unseen labels.
MathWriting 由五个不同的样本集组成，我们称之为“分割”：训练集、验证集、测试集、符号集和合成集。训练集、验证集和测试集仅包含人工书写的示例。符号集用于合成数据生成，在训练中不使用。训练集、验证集和测试集中的人工书写样本的划分部分基于作者，部分基于标签。更多细节请参见附录 K。实验表明，比书写风格更重要的是标签在训练过程中是否已被见过。这一事实也得到了组合泛化领域研究（keysers2020measuring）的支持。在已发表的版本中，验证集基于唯一归一化标签与训练集有 55%（8.5k 个样本）的交集，测试集有 8%的交集（647 个样本）。我们选择让训练集和测试集的交集较低，以便正确测量训练模型对未见标签的泛化能力。

3.4. Label Normalization  3.4.标签归一化

All samples in the dataset come with two labels: the LaTeX expression that was used during the data collection (annotation label in the InkML files), and a normalized version of it meant for model training, which is free from a few sources of confusions for an ML model (annotation normalizedLabel). 数据集中的所有样本都具有两个标签：数据收集过程中使用的 LTeX 表达式（InkML 文件中的标注标签），以及用于模型训练的标准化版本，它消除了对机器学习模型造成混淆的几个来源（标注标准化标签）。一个包含原始标签和标准化标签的示例在图 3 中提供。标签标准化涵盖三个主要类别（详细信息在附录 L 中提供）：

    variations used in print that can’t be reproduced in handwriting - e.g. bold, italic - or that haven’t been reproduced consistently by contributors.


    • 印刷中使用的变体，手写无法再现的，例如粗体、斜体，或贡献者未能一致再现的。

3.4.1. Limitations of normalization
3.4.1.归一化的局限性

The normalization process is purely syntactic, and can not cover cases where the meaning of the expression has to be taken into account. For example, a lot of expressions from Wikipedia use cos instead of \cos. It is often clear to a human reader whether the sequence of characters c,o,s represents the \cos command or simply three letters. However, this can not be reliably inferred by a syntactic parser, for example in tacos vs ta\cos. An alternative would be to update the raw labels, which we didn’t do because we wanted to keep the information that was used during the collection as untouched as possible. Similarly, cases like 10^{-1} usually mean {10}^{-1}, though they render exactly the same. We made the choice to normalize to the former because it’s the only option with a purely syntactic normalizer. It’s also better than not removing these extra braces because it gives more consistent label structures, which simplifies the model training problem.
归一化过程完全是句法的，不能涵盖需要考虑表达式含义的情况。例如，维基百科中的许多表达式使用 cos 而不是 \cos 。对于人类读者来说，字符序列 c,o,s 要么表示 \cos 命令，要么仅仅是三个字母，这通常很清楚。然而，句法解析器无法可靠地推断出这一点，例如在 tacos 与 ta\cos 的情况下。另一种选择是更新原始标签，但我们没有这样做，因为我们希望尽可能保持收集过程中使用的信息不变。类似地，像 10^{-1} 这样的情况通常表示 {10}^{-1} ，尽管它们渲染的结果完全相同。我们选择归一化到前者，因为这是唯一一个使用纯句法归一化器的选项。此外，与不删除这些额外的括号相比，这样做能提供更一致的标签结构，从而简化模型训练问题。

Synthetic samples were generated through the straightforward process of pasting inks of individual symbols (symbols) exactly where bounding boxes were located. This gives synthetic samples a very regular structure, see Figure 2. It is possible to improve this process by modifying the location, size or orientation of bounding boxes prior to generating the synthetic inks. This would soften LaTeX’s rigid structure and make synthetic data closer to human handwriting. Another application of these bounding boxes would be to bootstrap a recognizer that would also return character segmentation information. This kind of output is critical for some UI features - for example, editing an handwritten expression.
为合成样本提供边界框信息以及单个符号，以支持合成墨水生成的实验。合成样本通过将单个符号（符号）的墨水精确粘贴到边界框所在位置来生成。这使合成样本具有非常规则的结构，见图 2。通过在生成合成墨水之前修改边界框的位置、大小或方向，可以改进这一过程。这将使 LaTeX 的刚性结构得到软化，并使合成数据更接近人类手写。这些边界框的另一个应用是启动一个识别器，该识别器还会返回字符分割信息。这种输出对于某些 UI 功能至关重要——例如，编辑手写表达式。

MathWriting can also be improved by varying the label normalization. Changing it can have different benefits depending on the application, as mentioned above. We provide the source LaTeX string for that reason. Another possible improvement in recognition can come from additional contextual information, for instance the scientific field (Collier2022TransferAM) that can be added post-hoc. Combining recognizers with a language model (carbune2020fast) trained on a large set of mathematical expressions would be a step in a similar direction.
MathWriting 也可以通过改变标签归一化来改进。根据上文提到的，改变它可以带来不同的好处，具体取决于应用场景。我们提供原始的 LaTeX 字符串，原因就在于此。识别方面的另一个可能改进来自额外的上下文信息，例如科学领域（Collier2022TransferAM），这些信息可以在事后添加。将识别器与在大量数学表达式上训练的语言模型（carbune2020fast）结合，将是朝着类似方向迈出的一步。

7. Limitations  7.局限性

A single sample in MathWriting dataset has one handwritten LaTeX formula, see Figure 3. As a result, models that are trained on this dataset would perform poorly on complete handwritten documents, such as the IAMonDo dataset (IAMonDo). Also, as the dataset contains only LaTeX expressions, it is unlikely that models trained on it will accurately recognize handwritten text in English or other languages. As shown in Figure 4, some LaTeX tokens are way more frequent than others. Some infrequent tokens like \ni could be hard to recognise.
MathWriting 数据集中每个样本包含一个手写 LaTeX 公式，见图 3。因此，在 MathWriting 数据集上训练的模型在处理完整的文档时表现不佳，例如 IAMonDo 数据集（IAMonDo）。此外，由于该数据集只包含 LaTeX 表达式，因此训练在该数据集上的模型不太可能准确识别英文或其他语言的手写文本。如图 4 所示，某些 LaTeX 标记的频率远高于其他标记。一些不常见的标记，如 \ni ，可能难以识别。

8. Conclusion  8.结论

We introduced MathWriting, the largest dataset of online handwritten mathematical expressions to date, together with the experimental results of three different types of models. We hope this dataset will help advance research in both online and offline mathematical expression recognition. Additionally, we invite data practitioners to build on the dataset. We intentionally chose a file format for MathWriting close to the one used by CROHME to facilitate their combined use. We also provided original or intermediate representations (raw LaTeX strings, bounding boxes) to enable experimentation with the data itself, and suggested a few directions (Section 6.3).
我们介绍了 MathWriting，这是迄今为止最大的在线手写数学表达式数据集，并附带了三种不同类型模型实验的结果。我们希望这个数据集能够推动在线和离线数学表达式识别领域的研究进展。此外，我们邀请数据从业者基于此数据集进行开发。我们有意选择与 CROHME 使用的文件格式相近的格式，以便于它们能够结合使用。我们还提供了原始或中间表示（原始 LaTeX 字符串、边界框），以支持对数据本身进行实验，并提出了几个研究方向（第 6.3 节）。

Appendix C Postprocessing of MathWriting dataset
附录 CMathWriting 数据集的后处理

We applied no postprocessing to the collected inks other than dropping entirely those that were completely unreadable or had stray marks. Inks are provided in their original form, as they were recorded with the collection app. What we did not do was to discard samples that were very hard to read or ambiguous, because we believe this type of sample to be essential in training a high-quality model.
我们对收集到的墨迹没有进行任何后处理，除了完全无法辨认或带有杂乱标记的墨迹会被完全丢弃。墨迹以收集应用记录时的原始形式提供。我们没有丢弃那些非常难以辨认或模棱两可的样本，因为我们相信这类样本对于训练高质量模型至关重要。

Some cleanup was performed on the labels (ground truths). The goal was to make the dataset better suited to training an ML model, and eliminate unavoidable issues that occurred during the collection. After training some initial models, we manually reviewed samples for which they performed poorly. This helped identify a lot of unusable inks (near-blank, lots of stray strokes, scribbles, etc.), and a lot of ink/label discrepancies. A fairly common occurrence was a contributor forgetting to copy part of the prompted expression. We adjusted the label to what was actually written unless the ink contained a partially-drawn symbol, in which case we discarded the sample entirely. In this process we eliminated or fixed around 20k samples.
对标签（真实值）进行了一些清理。目标是使数据集更适合训练机器学习模型，并消除在收集过程中无法避免的问题。在训练一些初始模型后，我们手动审查了那些模型表现不佳的样本。这帮助我们识别出许多无法使用的墨迹（接近空白、大量杂乱的笔触、乱涂乱画等），以及许多墨迹/标签的不一致。一个相当常见的情况是贡献者忘记复制提示表达式的部分内容。如果墨迹中包含部分绘制的符号，我们会直接丢弃该样本；否则，我们会将标签调整为实际书写的内容。在这个过程中，我们删除或修复了大约 20k 个样本。

The most important postprocessing step was to normalize the labels: there are many different ways to write a mathematical expression in LaTeX format that will render to images that are equivalent in handwritten form. We applied a series of transformations to eliminate as many variations as possible while retaining the same semantic. This greatly improved the performance of models and made their evaluation more precise. We publish both the normalized and raw (unnormalized) labels, to enable people to experiment with other normalization procedures.
最重要的后处理步骤是对标签进行标准化：在 LaTeX 格式中，数学表达式有多种不同的写法，但它们渲染出的图像在手写形式上是等效的。我们应用了一系列转换来消除尽可能多的变化，同时保持相同的语义。这极大地提高了模型的性能，并使它们的评估更加精确。我们同时发布了标准化和原始（未标准化）的标签，以使人们能够尝试其他标准化方法。

This normalization is similar to what (deng2017imagetomarkup) did, but pushed further because of the specifics of handwritten MEs. See Section 3.4 for more detail.
这种归一化与(deng2017imagetomarkup)所做的工作相似，但由于手写数学表达式(MEs)的具体情况而更进一步。更多细节请参见第 3.4 节。

Appendix D Dataset split  附录 D 数据集划分

The valid and test splits are the result of multiple operations performed between 2016 and 2019. The first split operation, performed on the data available in 2016, was based on the contributor id: any given contributor’s samples would not appear in more than one split (either train, valid, test). This is common practice for handwriting recognition systems, to test how the recognizer performs on unseen handwriting styles.
有效集和测试集是 2016 年至 2019 年间执行多次操作的结果。第一次划分操作基于 2016 年可用的数据，以贡献者 ID 为依据：任何给定贡献者的样本不会出现在多个划分中（训练、有效、测试）。这是手写识别系统的一种常见做法，用于测试识别器对未见过的手写风格的性能。

Experiments then showed that a more important factor than the handwriting style was whether the label had already been seen during training. 实验表明，比手写风格更重要的是标签在训练过程中是否已被见过。随后的数据收集活动专注于增加标签的多样性，并将新样本添加到有效集和测试集中，这次以标签为划分依据：一个给定的标准化数学表达式不会出现在多个划分中。

Appendix H Sources of Noise
附录 H 噪声来源

The result of any task performed by humans will contain mistakes, and MathWriting is no exception. We’ve done our best to remove most of the mistakes, but we know that some remain.
人类执行任何任务的结果都会包含错误，MathWriting 也不例外。我们已尽力去除大部分错误，但我们知道仍有一些错误存在。

Stray strokes  杂乱笔画

These do not carry any meaning and should be ignored by any recognizer. Since they also appear in real applications, there could be some benefit in having some in the dataset to teach the model about them. That said, it being usually easier to add noise rather than to remove it, we made the choice of discarding as many inks containing stray strokes as possible. Not all inks with stray strokes have been found and removed though (e.g. train/9e64be65cb874902.inkml that was discovered post-publication). The fraction of inks containing stray strokes is significantly lower than 1%, and should not be an issue for training a model.
这些没有实际意义，任何识别器都应该忽略它们。由于它们也出现在实际应用中，在数据集中包含一些这样的样本可能有助于教会模型识别它们。不过，通常添加噪声比去除噪声更容易，因此我们选择了尽可能丢弃包含杂乱笔画的墨迹。尽管并非所有包含杂乱笔画的墨迹都被发现并移除（例如， train/9e64be65cb874902.inkml 是在发布后才发现的）。包含杂乱笔画的墨迹比例显著低于 1%，不应成为训练模型的问题。

Incorrect ground truth  错误的真值

Contributors did not always copy the prompt perfectly, leading to a variety of differences. In most of the cases we spotted, we were able to fix the label to match what had actually been written. A short manual review once the dataset was in its final state showed the rate of incorrect ground truth to be between 1% and 2%. Most of the mistakes are very minor, usually a single token added, missing or incorrect. Errors here also come from ambiguities or misuse of the LaTeX notation: expressions coming from Wikipedia contain some misuse like using \Sigma where \sum was more appropriate, \triangle instead of \Delta, \triangledown instead of \nabla, \begin{matrix}\end{matrix} instead of \binom, and also some handwriting-specific ambiguities like \dagger vs \top vs T. There are also some instances where reference numbers or extra punctuation are included.
贡献者并不总是完美地复制提示，导致出现各种差异。在我们发现的大多数情况下，我们能够修正标签以匹配实际书写内容。在数据集最终确定后进行的一次简短人工审查显示，错误真实标签的比率在 1%到 2%之间。大多数错误都非常小，通常是单个标记的添加、缺失或错误。这里的错误也来自 LaTeX 符号的模糊使用或误用：来自维基百科的表达式包含一些误用，如使用 \Sigma 而 \sum 更合适， \triangle 代替 \Delta ， \triangledown 代替 \nabla ， \begin{matrix}\end{matrix} 代替 \binom ，还有一些特定于手写的模糊性，如 \dagger 与 \top 与 T 。还有一些实例包含了参考数字或额外的标点符号。

Aggressive normalization  激进规范化

While the above sources of noise are unavoidable, normalization is a postprocessing operation that can in theory be tweaked to perfection. In practice, it’s a compromise between reducing accidental ambiguities (i.e. removing synonyms), and removing information. Examples: we made the choice of treating \binom as a synonym for a 2-element matrix. While it does improve recognition accuracy by making the problem easier, it also moves the burden of distinguishing between the two cases to downstream steps in the recognition pipeline. Similar things can be said about removing all commands that indicate that their content is text instead of math (e.g. \mbox), dropping size modifiers, rewriting function commands (e.g. \sin, \cos), etc. Using a different normalization could prove beneficial depending on the context the recognizer is used in practice. However, for the purpose of a benchmark any reasonable compromise is adequate.
虽然上述噪声来源不可避免，但归一化是一种后处理操作，理论上可以调整到完美。实际上，它是在减少偶然歧义（即删除同义词）和删除信息之间进行权衡。例如：我们选择将 \binom 视为 2 元素矩阵的同义词。虽然这通过使问题更容易来提高了识别精度，但也把区分这两种情况的负担转移到了识别流程的下游步骤。类似的情况也适用于删除所有表明其内容是文本而不是数学的命令（例如 \mbox ）、丢弃尺寸修饰符、重写函数命令（例如 \sin 、 \cos ）等。根据识别器在实际应用中的上下文，使用不同的归一化可能会带来好处。然而，对于基准测试的目的是任何合理的权衡都是足够的。

Appendix K Dataset split  附录 K 数据集划分

The valid and test splits are the result of multiple operations performed between 2016 and 2019. The first split operation, performed on the data available in 2016, was based on the contributor id: any given contributor’s samples would not appear in more than one split (either train, valid, test). This is common practice for handwriting recognition systems, to test how the recognizer performs on unseen handwriting styles.
有效集和测试集是多项操作在 2016 年至 2019 年间执行的结果。第一次划分操作是在 2016 年可用的数据上进行的，基于贡献者 ID：任何给定贡献者的样本不会出现在多个划分中（训练、有效、测试）。这是手写识别系统的一种常见做法，用于测试识别器在未见过书写风格上的表现。

Experiments then showed that a more important factor than the handwriting style was whether the label had already been seen during training. Subsequent data collection campaigns focused on increasing label variety, and new samples were added to valid and test, this time split by label: a given normalized mathematical expression would not appear in more than one split.
实验表明，比书写风格更重要的是标签在训练过程中是否已被见过。随后的数据收集活动专注于增加标签的多样性，并将新样本添加到验证集和测试集中，这次按标签进行划分：给定的标准化数学表达式不会出现在超过一个分割集中。

Appendix M Tokenization Code

Python code used in this work to tokenize LaTeX mathematical expressions.
本工作中用于对 LaTeX 数学表达式进行分词的 Python 代码。

%\begin{verbbox}
import re

_COMMAND_RE = re.compile(
  r’\\(mathbb{[a-zA-Z]}|begin{[a-z]+}|end{[a-z]+}|operatorname\*|[a-zA-Z]+|.)’)

def tokenize_expression(s: str) -> list[str]:
  tokens = []
  while s:
    if s[0] == ’\\’:
      tokens.append(_COMMAND_RE.match(s).group(0))
    else:
      tokens.append(s[0])

    s = s[len(tokens[-1]):]

  return tokens
